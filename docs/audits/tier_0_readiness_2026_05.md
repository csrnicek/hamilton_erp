### T0-FC-1 — Enable backup encryption

**Status: FAIL**

**Evidence:** No `encryption_key` set-config command appears anywhere in the codebase, install hooks, seed patches, or runbooks. `docs/RUNBOOK_BACKUP.md` pre-launch checklist (§11) lists offsite storage setup steps but does not include the `bench --site hamilton-erp.v.frappe.cloud set-config encryption_key '<random-key>'` step documented in the T0-FC-1 criterion itself. The T0-FC-1 item was identified 2026-04-30; no subsequent commit or runbook update reflects it being actioned.

**Remediation:**
1. Log into Frappe Cloud dashboard → Apps & Sites → hamilton-erp → Site Config.
2. Add key: `encryption_key` with a value generated by `python3 -c "import secrets; print(secrets.token_hex(32))"`.
3. Store the key immediately in 1Password under "Hamilton ERP — Frappe encryption_key". Never commit it to git.
4. Add a note in `docs/RUNBOOK_BACKUP.md` §11 that the encryption_key is set and its 1Password location.
5. All future backups taken after this point will be encrypted. Backups taken before are permanently unencrypted — this is acceptable in Phase 1 (no PII, cash-only), but must be done before Phase 2 card/PII data arrives.

### T0-FC-7 — Verify stock valuation method is FIFO at company level

**Status: FAIL**

**Evidence:** The install seed sets `Stock Settings.default_warehouse` (in `install.py:730–742`) but does not set `default_valuation_method`. No grep match for `valuation_method` or `FIFO` exists in any `.py` or `.json` file. The T0-FC-7 criterion notes that v16 made stock valuation method selectable per-company and that the Standard CoA default is FIFO — but this is an implicit reliance on ERPNext defaults, not an explicit Hamilton configuration. If an ERPNext minor release changes the default, or if a different install sequence produces a different default, Hamilton will silently move to Moving Average without any code change.

**Remediation:**
1. On `hamilton-erp.v.frappe.cloud` Desk, navigate to Stock Settings.
2. Confirm "Valuation Method" is FIFO (not Moving Average or LIFO).
3. Navigate to Company "Club Hamilton" → confirm no per-company override is set.
4. To make this explicit and durable: add to `install.py` `after_install`:
   ```python
   frappe.db.set_single_value("Stock Settings", "valuation_method", "FIFO")
   ```
5. Document in `docs/decisions_log.md` that Hamilton uses FIFO.
6. Estimated effort: 5 minutes verification + 15 minutes to harden in code.
